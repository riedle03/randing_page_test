Project Notes

How We Built This

이 페이지는 긴 문서를 한 번에 쏟아놓지 않고, 스크롤하면서 단계별로 이해할 수 있도록 다시 설계했습니다. 메인 랜딩 페이지와 같은 톤을 유지하면서 작업 흐름만 더 천천히 보여줍니다.

Pretendard + Card UI 명조체를 제거하고 프리텐다드 기반으로 더 또렷하고 현대적인 인상을 맞췄습니다.

01
Analyze the Reference

먼저 보는 법부터 맞췄습니다.

원본을 그냥 "예쁘다"로 지나가지 않고, 카드 비율, 헤드라인 크기, 고정 레이아웃, 스크롤 흐름처럼 구현에 바로 연결되는 정보로 해체했습니다.

What we looked for

색감, 여백, 카드 크기, 타이틀 위치, 등장 순서, 스크롤 리듬

Why it matters

감상보다 구조를 먼저 알아야 나중 수정도 흔들리지 않습니다.

02
Turn Taste Into Rules

느낌을 문서로 번역했습니다.

"애플 같다" 같은 인상만으로는 구현이 어렵기 때문에, PRD 안에서 수치와 규칙으로 다시 정리했습니다. 그래서 만들 때도, 고칠 때도 기준이 생겼습니다.

Core document

PRD.md에 구조, 타이밍, 크기, 색상, 반응형 기준을 모았습니다.

Result

같은 요청을 반복하지 않아도 일관된 결과를 유지할 수 있게 됐습니다.

03
Build the First Version

단일 파일로 빠르게 만들었습니다.

구현은 index.html 하나로 묶어서 비교 속도를 높였습니다. 구조가 단순할수록 눈으로 확인하고 다시 조정하기가 쉬웠습니다.

Assets

gradient-bg.svg, card-branding.svg, iphone-p-1080.jpeg

Build principle

라이브러리 없이 HTML, CSS, JS만으로 핵심 모션을 재현했습니다.

04
Compare the Gap

어디가 다른지 언어로 잡았습니다.

가장 중요한 순간은 비교 단계였습니다. 단순히 다르다고 느끼는 대신, 어느 구간에서 어떤 요소가 먼저 나오고 무엇이 어색한지를 구체적으로 짚었습니다.

Better feedback

첫 로드 인상, 카드 모핑, 폰 진입 타이밍처럼 사건 단위로 설명했습니다.

Why it works

차이를 정확히 말할 수 있어야 수정도 정확히 작동합니다.

05
Refine the Technique

마지막엔 기법까지 바꿨습니다.

카드가 화면에서 카드 형태로 수렴하는 장면은 transform보다 clip-path: inset()이 훨씬 자연스러웠습니다. 핵심 경험을 맞추기 위해 표현 방식 자체를 다시 선택했습니다.

What changed

요소를 움직이는 대신 보이는 영역을 줄이는 방식으로 모핑을 정리했습니다.

Final takeaway

완성도는 작은 차이 하나를 끝까지 다듬는 데서 나왔습니다.

One Experience

Same system, slower storytelling.

이번 수정에서는 폰트를 프리텐다드로 통일하고, 한 화면에 너무 많은 정보를 두지 않도록 전체 구조를 스크롤형 단계 카드로 다시 만들었습니다. 이제 이 페이지는 읽는 문서보다 따라가는 화면에 더 가깝습니다.
