Build Flow

이 프로젝트는 바로 코드를 쓰기보다 먼저 관찰하고, 차이를 언어화한 뒤, 구현과 수정을 반복하는 방식으로 진행했습니다.

Step 1

Reference Analysis

원본 사이트의 시각 구조를 해체해 색상, 카드 비율, 헤드라인 크기, 고정 레이아웃, 스크롤 구간별 애니메이션 흐름을 먼저 읽어냈습니다.

Step 2

PRD Drafting

"예쁜 느낌" 같은 모호한 표현 대신 수치와 규칙으로 정리해 재현 가능한 구현 문서로 바꿨습니다.

Step 3

First Build

단일 index.html 안에서 HTML, CSS, JS를 모두 구현해 비교 속도를 높이고 구조를 단순하게 유지했습니다.

Step 4

Gap Review

"다르다"가 아니라 "어느 구간에서 무엇이 어떻게 다르게 움직이는가"를 찾아내는 식으로 차이를 다시 정의했습니다.

Step 5

Refine the Technique

카드 진입 연출은 transform보다 clip-path: inset() 방식이 훨씬 안정적이어서, 최종 모핑 로직을 그 방향으로 정리했습니다.

Key Spec

메인 페이지의 감각을 만드는 핵심 규칙은 아주 적은 수의 결정에서 나왔습니다.

Visual Rules

  • 밝은 크림 톤 배경과 소프트한 그림자
  • 넓은 여백과 둥근 반경 중심의 카드 UI
  • 명조체 대신 시스템 산세리프 계열 타이포그래피
  • 큰 헤드라인, 짧은 문장, 절제된 보조 텍스트

Motion Rules

  • 모든 흐름은 스크롤 진행률에 따라 제어
  • 카드 축소와 이동은 하나의 연속된 사건처럼 보이게 구성
  • 타이틀과 오버레이는 카드 완성 시점에 맞춰 등장

What Mattered Most

비슷하게 보이게 만드는 데 중요한 것은 "요소를 나열하는 것"이 아니라 "어떤 순간에 어떤 인상이 생기는지"를 그대로 옮기는 것이었습니다.

01

Understand Before Building

보이는 결과만 흉내 내기보다, 왜 그런 구성이 선택됐는지 먼저 이해해야 다음 수정도 일관되게 이어질 수 있습니다.

02

Translate Taste Into Rules

"애플 같다"는 감상은 구현에 직접 쓰기 어렵습니다. 숫자, 여백, 레이어, 타이밍으로 번역해야 재현성과 품질이 생깁니다.

03

Refine the Smallest Gap

완성도는 마지막 10%의 차이에서 결정됩니다. 가장 어색한 구간 하나를 정확히 잡아내면 전체 인상이 빠르게 정리됩니다.

Same System, Different Page.

이 페이지는 별도의 블로그 스타일이 아니라 메인 랜딩 페이지의 확장 화면처럼 보이도록 다시 설계했습니다. 색상, 유리 질감 버튼, 둥근 카드, 타이포 스케일을 맞추고, 명조체는 제거해 하나의 제품 경험으로 연결했습니다.
